Resumen General

Esta unidad introduce los conceptos fundamentales de la programación. Se explica qué significa programar y cómo resolver problemas con una computadora siguiendo 5 pasos: comprender, analizar, diseñar el algoritmo, codificar y verificar. También se estudian los tipos de datos, los lenguajes de programación (máquina, bajo nivel, alto nivel) y los tipos de programación: secuencial, estructurada, lógica y orientada a objetos.

Conceptos Clave

Programar
Planificar y preparar de antemano las actividades para llevar a cabo una tarea o resolución de problemas.
Problema
El planteo de una situación a resolver mediante la aplicación de algún método.
Variable
Dato de entrada cuyo valor puede cambiar durante la resolución del problema.
Constante
Dato de entrada cuyo valor no cambia durante la resolución del problema.
Precondición
Restricción que indica qué valores válidos puede tomar un dato de entrada de tipo variable.
Pseudocódigo
Herramienta para diseñar soluciones: las instrucciones se escriben usando palabras en español.
Diagrama de Flujo
Representación gráfica de un algoritmo mediante símbolos y flechas.
Lenguaje Máquina
Lenguaje que entiende la computadora sin traducciones. Sus instrucciones son secuencias de 0 y 1.
Lenguaje de Alto Nivel
Instrucciones formadas por palabras similares al lenguaje natural. Independientes del procesador. Ej: C, Java, Python.
Compilador
Programa que traduce lenguaje de alto nivel a lenguaje máquina.
POO
Programación Orientada a Objetos: diseño basado en objetos que interactúan entre sí, con datos y acciones.
Programación Estructurada
Diseño modular descendente, usando las 3 estructuras básicas: secuencia, selección y repetición. Sin goto.

Explicación Detallada

¿Qué es programar?

Programar es planificar: preparar de antemano las actividades para resolver un problema. En esta materia se resuelven problemas usando algoritmos, estructuras de datos y lenguajes de programación.

Pasos para resolver un problema

  1. Comprender el problema: leer el enunciado hasta entender exactamente qué hay que resolver.
  2. Analizar el problema: identificar objetivo, datos de entrada (nombre, tipo, característica), precondiciones, datos de salida y lote de prueba.
  3. Diseñar el algoritmo: determinar el "cómo". Usar pseudocódigo o diagrama de flujo.
  4. Codificar: traducir el algoritmo a un lenguaje de programación.
  5. Verificar la solución: probar con datos normales, extremos y especiales usando la Prueba de Escritorio.

Análisis del Problema

Ejemplo de análisis completo

Problema: ingresar nombre y notas de 2 parciales, devolver VERDADERO si aprobó.

Objetivo: saber si el alumno aprobó.
Datos entrada: nombre (Cadena, Variable), notaParcial1 (Real, Variable), notaParcial2 (Real, Variable), CANTIDAD_EXAMENES (Entero, Constante = 2)
Precondiciones: notaParcial1 > 0 Y ≤ 10; notaParcial2 > 0 Y ≤ 10
Salida: alumno_aprobado (Lógico)

Lenguajes de Programación

  • Lenguaje Máquina: secuencias de 0 y 1. Muy difícil para personas. Ej: 0010 0000 0010 1000
  • Lenguaje de Bajo Nivel (ensamblador): usa códigos nemotécnicos. Dependiente del procesador. Ej: MOV R5, R6
  • Lenguaje de Alto Nivel: instrucciones similares al lenguaje natural. Independiente del procesador. Requiere compilador. Ej: C, Java, Python, C#, PHP.

Tipos de Programación

Programación Estructurada — Las 3 reglas

1. Diseño modular. 2. Cada módulo es diseñado de modo descendente. 3. Solo 3 estructuras básicas: secuencia, selección y repetición (no se permite goto).

  • Secuencial: instrucciones en orden, usa goto para bifurcaciones.
  • Estructurada: modular, descendente, sin goto.
  • Lógica: define hechos y relaciones (base de conocimiento).
  • Orientada a Objetos (POO): objetos con datos y acciones que interactúan.
