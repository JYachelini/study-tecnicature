Resumen General

Vista general de la unidad

La Unidad 5 introduce las estructuras de repetición (ciclos), que permiten ejecutar un bloque de instrucciones múltiples veces sin duplicar código. Se estudian dos tipos: el ciclo While ("Mientras"), que repite mientras una condición sea verdadera y puede no ejecutarse si la condición empieza en falso; y el ciclo For ("Para"), para iteraciones exactas cuando se conoce de antemano la cantidad de repeticiones. Se distinguen ciclos con contador fijo de los inexactos con valor de corte, donde el primer dato debe leerse antes del ciclo y repetirse al final de cada iteración. Finalmente se estudia el anidamiento: estructuras dentro de otras, donde las iteraciones totales se multiplican.

Conceptos Clave

Términos y definiciones del material

Estructura de Repetición
Genera iteraciones donde ciertas instrucciones se ejecutan una cantidad determinada de veces, evitando duplicar código.
Ciclo / Iteración
Cada ejecución del bloque de instrucciones. Sinónimo de "vuelta" o "repetición".
While — Mientras
Repite mientras la condición sea verdadera. Puede no ejecutarse si la condición es falsa desde el inicio.
For — Para
Ciclo de iteraciones exactas. Lleva la variable de control y la incrementa automáticamente.
Contador
Variable entera que registra cuántas veces se ejecutó el ciclo. En While se incrementa manualmente; en For, automáticamente.
Ciclo Inexacto
Ciclo While donde no se conoce la cantidad de iteraciones de antemano; depende de lo que ingrese el usuario.
Valor de Corte
Valor especial que indica el fin del ciclo (ej: -1 para edades, "Fin" para nombres).
Paso / Incremento
Valor en que aumenta el contador del For en cada iteración. Por defecto es 1, pero puede cambiarse.
Variable de Índice
Variable entera (i, j…) que controla las iteraciones del ciclo For.
Anidamiento
Colocar una estructura de repetición dentro de otra. Las iteraciones totales se calculan multiplicando ambos contadores.
Bucle Infinito
Error causado por no modificar la condición del While, haciendo que nunca se vuelva falsa.
Límites Inclusivos
En el For, tanto el valor inicial como el final son válidos y se ejecutan.

♻ ¿Por qué necesitamos ciclos?

Hasta ahora, cada instrucción se ejecutaba como máximo una vez. Si necesitamos verificar 10 o 30 edades, copiar el mismo bloque repetidamente suma líneas sin resolver nada más complejo.

La solución son las estructuras iterativas, de repetición o ciclos: generan iteraciones donde ciertas instrucciones se ejecutan la cantidad de veces necesaria.

💡 "La repetición es la madre de la habilidad" — frase con la que abre el material.

▶ Ciclo While — Mientras... Hacer

Repite un bloque mientras se cumpla una condición específica.

while (condicion) {
    instrucciones
}

Funcionamiento:

  1. Se evalúa la condición entre paréntesis.
  2. Si es verdadera, entra y ejecuta las instrucciones.
  3. Al terminar la iteración, vuelve a evaluar la condición.
  4. Cuando es falsa, el bucle finaliza.
⚠

Si la condición es falsa desde el principio, el bucle nunca se ejecuta (cero iteraciones).

Tipo 1 — Con contador (iteraciones conocidas):

int contador = 0;
while (contador < 10) {
    cout << "Ingrese la edad";
    cin >> edad;
    if (edad >= 18) cout << "Mayor de edad";
    else            cout << "Menor de edad";
    contador = contador + 1; // ¡Obligatorio!
}
⚠

Omitir el incremento del contador causa un bucle infinito.

Tipo 2 — Con valor de corte (iteraciones desconocidas):

cout << "Ingrese edad (-1 para salir): ";
cin >> edad;                     // ← Primer lectura ANTES del ciclo
while (edad >= 0) {
    if (edad >= 18) cout << "Mayor";
    else            cout << "Menor";
    cout << "Ingrese edad (-1 para salir): ";
    cin >> edad;                 // ← Al FINAL de cada iteración
}
📌

El primer dato se pide antes del ciclo y se vuelve a pedir al final de cada iteración. Así el valor de corte no se procesa.

▶ Ciclo For — Para... Hacer

Para iteraciones exactas. La variable de control está incorporada y se incrementa automáticamente.

for (int i = 0; i < N; i++) {
    instrucciones
}
// Las tres partes:
// 1) int i = 0   → inicialización (una sola vez)
// 2) i < N       → condición de corte (antes de cada iteración)
// 3) i++         → incremento automático (al final de cada iteración)
🚫

Regla de oro: Nunca incrementar i manualmente dentro del cuerpo del For. La estructura ya lo hace; hacerlo produce saltos inesperados.

Paso personalizado — pares del 2 al 12:

for (int i = 2; i <= 12; i = i + 2) {
    cout << "Par: " << i;
}
// Imprime: 2 4 6 8 10 12 — sin if interno

▶ Estructuras Anidadas

Por cada iteración del bucle externo se ejecutan todas las iteraciones del bucle interno.

Total = iteraciones_externo × iteraciones_interno

For/For — 30 alumnos, 4 notas cada uno:

for (int i = 0; i < 30; i++) {     // 30 alumnos
    prom = 0;
    for (int j = 0; j < 4; j++) { // 4 notas por alumno
        cin >> nota;
        prom = prom + nota;
    }
    cout << prom / 4;
}
// Total: 30 × 4 = 120 iteraciones internas

For/While — cantidad de alumnos desconocida:

cin >> nombre;
while (nombre != "Fin") {           // While: cantidad desconocida
    prom = 0;
    for (int j = 0; j < 4; j++) { // For: siempre 4 notas
        cin >> nota;
        prom = prom + nota;
    }
    cout << prom / 4 << endl;
    cin >> nombre; // releer al final
}
While externo cuando no se sabe cuántos alumnos hay; For interno porque siempre son exactamente 4 notas.
