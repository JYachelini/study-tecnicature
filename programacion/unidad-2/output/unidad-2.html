Resumen General

Esta unidad profundiza el concepto de variable: un espacio en memoria RAM al que el programa le asocia un nombre para leer y modificar su valor. Se estudian las reglas para nombrar variables (incluyendo camelCase), los tipos de datos, las diferencias entre variables y constantes, y los tres tipos de operadores: aritméticos (+, -, *, /, %), relacionales (>, <, ==, !=) y lógicos (Y, O, NO).

Conceptos Clave

Variable
Posición en memoria RAM a la que se asocia un nombre para leer y/o modificar su valor.
Constante
Valor que no cambia durante la ejecución del programa. Se define una vez y no se modifica.
Tipo de dato
Define el rango de valores posibles y las operaciones que se pueden hacer sobre una variable.
camelCase
Convención para nombres de variables: primera letra minúscula, cada palabra siguiente empieza en mayúscula. Ej: cuentaBancaria.
Definir variable
Declarar la variable antes de usarla. En pseudocódigo: "Definir nombre como Tipo;"
Operadores Aritméticos
+, -, *, /, % (módulo/resto). Realizan cálculos matemáticos sobre valores numéricos.
Operadores Relacionales
>, <, >=, <=, ==, !=. Comparan dos valores y devuelven Verdadero o Falso.
Operadores Lógicos
Y (AND), O (OR), NO (NOT). Combinan expresiones lógicas. Devuelven Verdadero o Falso.
Asignación
Operación que guarda un valor en una variable. En pseudocódigo: nombre ← valor o nombre = valor.

Explicación Detallada

¿Qué es una Variable?

Una variable es una posición en memoria RAM a la que desde nuestro programa le asociamos un nombre para poder accederla, leer y/o modificar su valor. Toda variable tiene un tipo de dato y un nombre (identificador).

Reglas para nombrar una variable

  1. Debe comenzar con una letra (a-z) o el carácter _ (guión bajo).
  2. Luego del primer carácter solo puede contener letras, números o _ .
  3. No puede contener espacios en blanco ni caracteres especiales (?, -, etc.).

La convención camelCase es la más usada: primera letra minúscula, cada palabra siguiente empieza con mayúscula.

Operadores Lógicos

Tablas de verdad

Y (AND): Verdadero solo si AMBAS condiciones son verdaderas.
O (OR): Verdadero si AL MENOS UNA condición es verdadera.
NO (NOT): Invierte el valor lógico.

Ejemplo: nota >= 4 Y nota <= 6 → Verdadero solo si nota está entre 4 y 6.
