Resumen General

Un algoritmo es una secuencia finita de instrucciones que describen con precisión las operaciones que una computadora debe realizar para resolver un problema en tiempo finito. Son independientes del lenguaje y del hardware. Para diseñarlos se usan pseudocódigo y diagrama de flujo. La técnica de diseño descendente ("divide y vencerás") divide el problema en subproblemas más simples.

Características fundamentales de un algoritmo

PRECISO
Indica el orden de realización de cada paso.
NO AMBIGUO
Si se sigue 2 veces, se alcanza el mismo resultado.
FINITO
Tiene un número finito de pasos. Siempre termina.

Conceptos Clave

Algoritmo
Secuencia finita de instrucciones precisas que describen cómo resolver un problema en tiempo finito.
Pseudocódigo
Representación textual de un algoritmo usando palabras en español e instrucciones estructuradas. No es ejecutable directamente.
Diagrama de Flujo
Representación gráfica de un algoritmo. Usa figuras geométricas conectadas con flechas.
Diseño Descendente
Técnica de dividir el problema en subproblemas cada vez más simples hasta poder codificarlos. "Divide y vencerás".
Refinamiento Sucesivo
Cada paso del algoritmo se va detallando progresivamente hasta llegar a instrucciones implementables.
Entrada del algoritmo
Los datos necesarios para resolver el problema. Deben estar definidos antes del procesamiento.
Proceso del algoritmo
Los pasos o instrucciones que transforman los datos de entrada en la salida esperada.
Salida del algoritmo
El resultado esperado tras ejecutar todos los pasos del proceso.
Instrucción Mostrar
Instrucción de pseudocódigo que muestra un valor o mensaje por pantalla.
Instrucción Leer
Instrucción de pseudocódigo que lee un valor ingresado por el usuario y lo guarda en una variable.

Explicación Detallada

¿Qué es un algoritmo?

Un algoritmo es un método para resolver un problema. Más precisamente:

Definición formal [Knuth, 1968]

Secuencia finita de instrucciones, reglas o pasos que describen en forma precisa las operaciones que una computadora debe realizar para llevar a cabo una tarea en tiempo finito.

Los algoritmos son independientes tanto del lenguaje de programación como de la computadora. Como una receta de cocina escrita en distintos idiomas: los pasos siguen siendo los mismos.

Las 3 partes de un algoritmo

  1. Datos de entrada: los datos necesarios para resolver el problema.
  2. Proceso: los pasos a ejecutar (las instrucciones).
  3. Salida esperada: el resultado que se quiere obtener.

Ejemplo de algoritmo en pseudocódigo

Proceso VerificarPedido Leer pedido; Leer deudaCliente; Si ( deudaCliente == Falso ) Entonces Mostrar "Pedido aceptado"; Sino Mostrar "Pedido rechazado"; Fin Si FinProceso

Diseño Descendente ("Divide y vencerás")

La técnica consiste en dividir el problema en subproblemas más simples, y a su vez dividir esos subproblemas en otros aún más simples, hasta llegar a pasos que puedan implementarse directamente.

Analogía

Mudarse = buscar nuevo hogar + adquirir la propiedad + realizar la mudanza.
Buscar nuevo hogar = visitar inmobiliarias + navegar sitios web + comparar propiedades + visitar propiedades + decidir.

Refinamiento Sucesivo

El primer planteo del algoritmo tendrá pocos pasos generales. Se debe refinar cada paso dividiéndolo en pasos más pequeños y detallados, hasta alcanzar instrucciones que puedan volcarse a una computadora. Este proceso se llama refinamiento sucesivo.

Pseudocódigo vs. Diagrama de Flujo

PseudocódigoDiagrama de Flujo
Representación textual con instrucciones en españolRepresentación gráfica con formas y flechas
Más fácil de escribir y modificarMás visual, fácil de seguir el flujo
Cercano al código realMejor para comunicar el algoritmo
