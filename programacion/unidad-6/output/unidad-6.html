Funciones y Procedimientos

Modularización · Top Down · Parámetros · Variables Locales y Globales

Resumen General

La modularización (Desarrollo Top Down) consiste en dividir el problema en subproblemas más pequeños y resolverlos mediante funciones y procedimientos. Las funciones retornan un valor; los procedimientos no retornan. Ambos reciben parámetros. Las variables definidas dentro de un módulo son locales (no accesibles desde afuera). Esta técnica simplifica el código, permite la reutilización y mejora la legibilidad.

Función
✓ Recibe parámetros
✓ Realiza una tarea
✓ Retorna un valor
Ej: calcularRaiz(n)
Procedimiento
✓ Recibe parámetros
✓ Realiza una tarea
✗ No retorna valor
Ej: mostrarResultado(r)

Conceptos Clave

Modularización
Dividir un programa en módulos más pequeños, cada uno con una tarea específica. "Divide y vencerás".
Desarrollo Top Down
Metodología de ir del problema general a subproblemas específicos, descomponiéndolos progresivamente.
Módulo / Subprograma
Unidad de código independiente con nombre, que puede llamarse desde el programa principal. Sinónimos: rutina, subrutina, función, procedimiento.
Función
Módulo que realiza una tarea y RETORNA un valor al código que la llamó.
Procedimiento
Módulo que realiza una tarea pero NO retorna ningún valor. En C++: función void.
Parámetro
Dato que se le pasa a una función o procedimiento para que trabaje con él. Define la "interfaz" del módulo.
Variable Local
Variable definida dentro de un módulo. Solo existe y es accesible dentro de ese módulo.
Variable Global
Variable definida fuera de todos los módulos. Accesible desde cualquier parte del programa.
Reutilización de Código
Un módulo ya escrito puede usarse múltiples veces en distintos programas sin reescribirlo. Evita "reinventar la rueda".
Algoritmo Principal
El módulo raíz que llama a los subalgoritmos para resolver el problema completo.

Explicación Detallada

¿Por qué modularizar?

A medida que los algoritmos se vuelven más complejos, necesitamos una estrategia para manejar esa complejidad. La modularización resuelve esto dividiendo el problema en partes más manejables.

Analogía de la mudanza

Mudarse = buscar hogar + adquirir propiedad + realizar mudanza.
Buscar hogar = visitar inmobiliarias + comparar propiedades + decidir cuál adquirir.

Esto es el Desarrollo Top Down: dividir el problema grande en subproblemas cada vez más pequeños.

Ventajas de la Modularización

  • Simplifica el programa: cada módulo es más simple de programar que el problema completo.
  • Permite la reutilización: un módulo escrito una vez puede usarse en otros programas.
  • Más legible: el algoritmo principal es una lista corta de llamadas a módulos, fácil de leer.
  • Más fácil de probar: cada módulo puede probarse de forma independiente.
  • Trabajo en equipo: distintos módulos pueden ser desarrollados por distintas personas.

Función vs. Procedimiento en Pseudocódigo

// FUNCIÓN: retorna un valor Real calcularPromedio(Real nota1, Real nota2) Definir promedio como Real; promedio = (nota1 + nota2) / 2; Retornar promedio; FinFunción // PROCEDIMIENTO: no retorna valor mostrarSaludo(Cadena nombre) Mostrar "Hola, ", nombre; FinProcedimiento // PROGRAMA PRINCIPAL que los usa: Proceso Principal Definir prom como Real; prom = calcularPromedio(8, 6); // llama a la función mostrarSaludo("Ana"); // llama al procedimiento FinProceso

Variables Locales vs. Globales

Variable Local
• Definida dentro de un módulo
• Solo existe durante la ejecución del módulo
• No puede accederse desde afuera
• Recomendada: evita efectos secundarios
Variable Global
• Definida fuera de todos los módulos
• Existe durante toda la ejecución
• Accesible desde cualquier módulo
• Usar con cuidado: puede causar errores difíciles de rastrear
