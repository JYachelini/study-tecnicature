Funciones y Procedimientos

Modularización · Top Down · Parámetros · Scope · Recursividad

📌 Material 📄 clase_6-_FUNCIONES_Y_PROCEDIMIENTOS.pdf

Resumen General

La modularización (Desarrollo Top Down) divide un problema complejo en subproblemas más simples llamados módulos. En C++ existen dos tipos: las funciones (retornan un único valor) y los procedimientos (tipo void, pueden retornar cero o varios valores por referencia). La comunicación entre módulos se hace mediante parámetros: por valor (copia) o por referencia (símbolo &, acceso directo). Las variables dentro de un módulo son locales y desaparecen cuando el módulo termina. Un caso especial de función es la recursividad: una función que se llama a sí misma hasta alcanzar un caso base.

Función
✓ Recibe parámetros de entrada
✓ Realiza una tarea
✓ Retorna exactamente un valor
✓ Debe terminar con return valor;
Ej: int max(int a, int b)
Procedimiento (void)
✓ Recibe parámetros
✓ Realiza una tarea
✓ Puede modificar variables por &
✗ No retorna valor directo
Ej: void dividir(int a, int b, int &c, int &r)
Pasaje por Valor
• El módulo recibe una copia
• Cambios internos NO afectan al original
• Parámetro de entrada
Ej: void mostrar(float promedio)
Pasaje por Referencia
• El módulo recibe la dirección real
• Cambios internos SÍ afectan al original
• Parámetro de salida (usa &)
Ej: void leer(int &nota1, int &nota2)

Conceptos Clave

Modularización
Dividir un programa complejo en módulos más pequeños. Cada uno hace una sola tarea. "Divide y vencerás".
Desarrollo Top Down
Metodología: partir del problema general y descomponerlo iterativamente en subproblemas cada vez más simples.
Módulo / Subprograma
Unidad de código independiente con nombre propio. Sinónimos: rutina, subrutina, función, procedimiento.
Función
Módulo que ejecuta una tarea y RETORNA exactamente un valor. Siempre termina con return valor;.
Procedimiento (void)
Módulo que realiza una tarea pero no retorna un valor directo. Puede devolver resultados modificando parámetros por referencia.
Firma de la Función
Combinación del nombre y la lista de parámetros. Ej: max(int num1, int num2). Identifica unívocamente a la función.
Parámetro vs Argumento
Parámetro: variable en la declaración de la función. Argumento: valor real que se pasa al llamarla. max(a, b) → a y b son argumentos.
Pasaje por Valor
El módulo recibe una COPIA del argumento. Los cambios internos NO afectan a la variable original.
Pasaje por Referencia
El módulo recibe la dirección real de la variable. Los cambios internos SÍ modifican la variable original. Se declara con &.
Variable Local
Variable definida dentro de un módulo. Solo existe mientras el módulo se ejecuta; desaparece al terminar.
Variable Global
Variable definida fuera de todos los módulos. Existe durante toda la ejecución. No debe accederse directamente desde módulos.
Prioridad de Nombre
Si una variable local y una global tienen el mismo nombre, dentro del módulo siempre se usa la LOCAL.
Abstracción
"Caja negra": importa qué recibe y qué devuelve el módulo, no cómo lo hace internamente. Permite trabajar con módulos sin conocer su implementación.
Ocultamiento de información
Las variables y operaciones internas de un módulo quedan protegidas del resto del programa. Consecuencia de la abstracción.
Recursividad
Cuando una función se llama a sí misma. Requiere siempre un caso base (condición de corte) y una ecuación de recurrencia.
Caso Base
Condición que detiene la recursividad. Sin caso base, la función se llama infinitamente (desbordamiento de pila).
Reutilización de Código
Un módulo bien escrito puede usarse en distintos programas sin reescribirlo. Clave: nombres genéricos en parámetros.
Declaración anticipada
Escribir la firma de la función antes del main para que el compilador la conozca, y el cuerpo completo después.

Explicación Detallada

1. ¿Por qué modularizar?

A medida que los algoritmos crecen, manejar toda la lógica en un único bloque se vuelve inmanejable. La modularización (o Desarrollo Top Down) resuelve esto:

Analogía de la mudanza

Mudarse = buscar hogar + adquirir propiedad + realizar mudanza.
Buscar hogar = visitar inmobiliarias + navegar la web + comparar y decidir.
Cada subproblema, a su vez, puede seguir dividiéndose hasta ser trivial.

Beneficios concretos:

  • Simplifica: subproblemas pequeños son más fáciles de programar.
  • Reutilización: un módulo escrito se puede usar en otros programas.
  • Legibilidad: el main queda como una lista corta de llamadas.
  • Testing independiente: cada módulo puede probarse por separado.
  • Mantenimiento localizado: un cambio o arreglo afecta solo al módulo correspondiente.

2. Abstracción y Caja Negra

Al dividir en módulos nace un concepto clave: la abstracción. Trabajamos con los módulos como cajas negras: solo nos importa qué datos reciben y qué devuelven, no cómo lo hacen internamente. Cada módulo debe ser independiente y realizar una sola tarea.

Regla de oro: No mezcles responsabilidades en un mismo módulo. Separá la obtención de datos (procedimiento) del cálculo (función pura).

3. Tipos de Módulos en C++

Función
• Tipo de retorno antes del nombre
• Siempre termina con return valor;
• Retorna exactamente un valor
Procedimiento (void)
• Tipo de retorno: void
• Puede usar return; solo para salir antes
• Devuelve resultados por referencia (&)
// Función: retorna el máximo (ejemplo del apunte) int max(int num1, int num2) { int result; if (num1 > num2) result = num1; else result = num2; return result; } // Procedimiento: divide y entrega cociente y resto por referencia void dividir(int dividendo, int divisor, int &cociente, int &resto) { cociente = dividendo / divisor; resto = dividendo % divisor; }

4. Firma de la Función y Declaración Anticipada

La firma es la combinación del nombre + lista de parámetros. Para que el main pueda llamar a una función, el compilador debe conocerla antes. Opciones:

  • Escribir la función completa antes del main.
  • Declarar solo la firma antes del main y escribir el cuerpo después.
int max(int num1, int num2); // solo la firma — declaración anticipada int main() { int ret = max(100, 200); return 0; } int max(int num1, int num2) { // cuerpo completo después return (num1 > num2) ? num1 : num2; }

5. Ámbito de Variables (Scope)

Variable Local
• Definida dentro del módulo
• Almacenada en su propio ámbito de memoria
• Solo accesible dentro del módulo
• Desaparece cuando el módulo termina
Variable Global
• Definida fuera de todos los módulos
• Existe durante toda la ejecución
• Visible desde cualquier parte
• Nunca accederla directamente desde un módulo
Regla de prioridad de nombre: Si existe una variable local con el mismo nombre que una global, dentro del módulo siempre se usa la LOCAL. La global queda "tapada".
Regla de buenas prácticas: Acceder a una variable global directamente desde un módulo rompe la abstracción. La forma correcta es pasarla como parámetro.

6. Pasaje de Parámetros

Por Valor: el módulo recibe una copia. Cambios internos no se reflejan afuera.

void mostrarPromedio(float promedio) { // recibe copia promedio = 0; // no afecta a la variable original cout << promedio; }

Por Referencia: el módulo recibe la dirección real de la variable. Los cambios sí modifican el original.

void obtenerNotas(int &nota1, int &nota2) { // recibe referencia cin >> nota1 >> nota2; // modifica las variables originales }

Combinando entrada y salida:

// bool como retorno para indicar éxito/fallo + resultados por referencia bool dividirSeguro(int dividendo, int divisor, int &cociente, int &resto) { if (divisor == 0) return false; // fallo cociente = dividendo / divisor; resto = dividendo % divisor; return true; // éxito }

7. Consejos de Modularización

  • Maximizar la abstracción: usá nombres genéricos. promedio(int val1, int val2) es reutilizable para cualquier promedio. promedioNotas(int nota1, int nota2) solo sirve para notas.
  • Una sola tarea por módulo: no mezcles "pedir datos al usuario" con "calcular". Creá un procedimiento para capturar y una función pura para calcular.

Recursividad

Una función es recursiva cuando se llama a sí misma. Funciona como un bucle, pero sin usar for o while. Las llamadas se "apilan" en memoria hasta que se cumple el caso base, momento en que el control empieza a volver hacia atrás hasta llegar al main.

Partes obligatorias

Caso Base
• Condición que detiene la recursividad
• No depende de llamadas anteriores
• Sin él: desbordamiento de pila
Ecuación de Recurrencia
• La función se llama a sí misma
• Parámetros modificados para acercarse al caso base
• Reduce el espacio del problema en cada vuelta

Factorial (n!)

Caso base: 0! = 1 (si n es 0, retorna 1). Ecuación: n * factorial(n-1).

int factorial(int n) { if (n == 0) return 1; // caso base return n * factorial(n - 1); // ecuación de recurrencia } // Traza: factorial(4) // → 4 * factorial(3) // → 3 * factorial(2) // → 2 * factorial(1) // → 1 * factorial(0) → 1 (caso base) // Resultado: 4*3*2*1 = 24

Potencia (an)

Caso base: a^0 = 1 (si exponente es 0, retorna 1). Ecuación: base * potencia(base, exponente - 1).

int potencia(int base, int exp) { if (exp == 0) return 1; // caso base return base * potencia(base, exp - 1); // ecuación de recurrencia }

Sucesión de Fibonacci

Dos casos base: Fib(0) = 1 y Fib(1) = 1. Ecuación: Fib(n-1) + Fib(n-2).

int fib(int n) { if (n == 0 || n == 1) return 1; // casos base return fib(n - 1) + fib(n - 2); // ecuación de recurrencia } // Secuencia: 1, 1, 2, 3, 5, 8, 13, 21, ...
Clave para identificar recursividad: buscá el caso base (condición de corte) y verificá que la ecuación de recurrencia siempre "achica" el problema acercándose a ese caso base. Si no se acerca, habrá recursión infinita.
